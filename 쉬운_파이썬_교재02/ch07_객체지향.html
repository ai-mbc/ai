CHAPTER 07

객체지향 프로그래밍

지금까지 메뉴 '가격'은 딕셔너리에, '계산'은 함수에 따로따로 있었어요. 그런데 메뉴 하나에는 이름·가격·재고·조리법이 다 딸려 있죠. 이 데이터와 기능을 한 덩어리('객체')로 묶는 게 객체지향이에요. 메뉴를 진짜 '물건'처럼 다룰 수 있게 됩니다.

🍜 파이 분식 이야기 — 7화. 메뉴가 '살아 있는 물건'이 되다

메뉴가 많아지자 관리가 흩어졌어요. 김밥의 가격은 여기, 재고는 저기, 조리법은 또 다른 파일에… 사장님은 생각합니다. "김밥이라는 물건 하나에 가격·재고·조리법을 통째로 붙여 두면 어떨까?" 이 아이디어가 객체지향이에요. 먼저 '메뉴 설계도(클래스)'를 하나 그려 두면, 김밥·라면·떡볶이를 그 설계도로 찍어내듯(객체) 만들 수 있어요. 게다가 '음료 메뉴'는 '메뉴 설계도'를 물려받아(상속) 조금만 고쳐 쓰면 됩니다.

🎯 이 장을 끝내면 이런 걸 할 수 있어요

  • 클래스와 객체(인스턴스)의 관계를 설명할 수 있다
  • 생성자(__init__)로 객체를 초기화할 수 있다
  • self의 의미를 이해하고 메서드를 만들 수 있다
  • 상속으로 코드를 재사용하고 super()를 쓸 수 있다
  • 재정의(오버라이딩)와 정적/클래스 메서드를 활용할 수 있다
🏗️ 객체지향(OOP)이란?
프로그램을 '객체'라는 부품 단위로 나눠 만드는 방식이에요. 각 객체는 데이터(상태)와 기능(행동)을 함께 가져요. 현실 세계를 그대로 코드로 옮기기 좋아서 큰 프로그램을 체계적으로 만들 수 있습니다.

1클래스와 객체

📐 클래스(Class)
객체를 만들기 위한 틀(설계도, 붕어빵 틀)이에요. "이런 데이터와 기능을 가진다"는 청사진입니다.
🎁 객체(Object) = 인스턴스(Instance)
클래스라는 틀로 실제로 찍어낸 결과물이에요. 하나의 클래스로 여러 객체를 만들 수 있어요.
🐟 비유 — 클래스는 '메뉴 설계도', 객체는 그 설계도로 찍어낸 김밥·라면·떡볶이 각각이에요. 같은 설계도(이름·가격·재고를 가진다)에서 나왔지만, 김밥은 3500원·라면은 4000원처럼 각자 다른 값을 담습니다. 붕어빵 틀 하나로 팥·슈크림 붕어빵을 굽는 것과 똑같죠.

객체는 두 가지를 가져요.

구분다른 이름들뜻
명사적 특성데이터·변수·필드·속성(Attribute)객체가 가진 값 (예: 이름, 나이)
동사적 특성함수·메서드(Method)·기능객체가 하는 일 (예: 인사하기)
python
class Menu:                   # '메뉴 설계도' 정의 (첫 글자 대문자)
    def 안내(self):
        print("파이 분식 메뉴입니다!")

김밥 = Menu()    # 설계도로 '김밥' 메뉴 객체 찍어내기
김밥.안내()       # 객체의 메서드 호출 (점 . 은 참조 연산자)
파이 분식 메뉴입니다!
💡 카멜 표기법
클래스 이름은 관례적으로 첫 글자를 대문자로 써요(Person, BankAccount). 변수·함수(소문자)와 구분하기 위해서예요.

2생성자와 소멸자

🔨 생성자 __init__()
객체가 만들어질 때 자동으로 실행되는 함수예요. 보통 여기서 객체의 초기 데이터를 설정합니다.
🏭 비유 — 메뉴를 등록하는 순간 "이 메뉴 이름은 뭐고, 가격은 얼마야?"를 채워 넣는 과정이 생성자예요. 메뉴 객체가 태어나는 순간 딱 한 번 실행돼, 각 메뉴에 고유한 이름·가격을 새깁니다.
python
class Menu:
    def __init__(self, 이름, 가격):    # 생성자
        self.이름 = 이름       # 이 메뉴만의 데이터 초기화
        self.가격 = 가격

김밥 = Menu("김밥", 3500)     # 만드는 순간 이름·가격이 채워짐
print(김밥.이름, 김밥.가격)
김밥 3500
⚠️ 생성자는 하나만!
파이썬은 생성자를 여러 개 만들 수 없어요. 같은 클래스에 __init__을 두 번 정의하면 마지막 것만 유효합니다.
💥 소멸자 __del__()
객체가 사라질 때 자동 실행되는 함수예요. 객체를 참조하는 곳이 하나도 없어지면(레퍼런스 카운트 0) 소멸됩니다.

객체를 없애는 방법은 세 가지예요: ① 변수에 새 객체를 대입   ② None 대입   ③ del 변수

💡 레퍼런스 카운트
객체를 가리키는 이름표가 몇 개인지 세는 숫자예요. p2 = p1이면 카운트 2, 둘 다 없어지면 0이 되어 소멸자가 실행됩니다.

3self와 메서드

👉 self
메서드의 첫 번째 인자로, "이 객체 자신"을 가리켜요. 객체의 데이터에 접근할 때 self.변수 형태로 씁니다.
🪞 비유 — "나 자신"을 가리키는 손가락이에요. "내 이름", "내 나이"라고 할 때의 그 '내(self)'입니다.
python
class Person:
    def __init__(self, name):
        self.name = name          # 이 객체의 name

    def print_info(self):         # 인스턴스 메서드는 self가 필수
        print(f"제 이름은 {self.name}입니다")

p1 = Person("홍길동")
p1.print_info()   # self는 자동으로 p1이 전달됨
제 이름은 홍길동입니다
⚠️ self를 빼먹지 마세요
인스턴스 메서드의 첫 인자는 반드시 self여야 해요(이름은 바꿀 수 있지만 관례상 self). 호출할 땐 p1.print_info()처럼 self 자리는 자동으로 채워집니다.
💡 이름을 찾는 순서 (네임스페이스)
객체에서 변수를 찾을 땐 ① 인스턴스 → ② 클래스 → ③ 전역 순서로 뒤져요. 그래서 클래스 변수(공유값)와 인스턴스 변수(각자값)를 함께 쓸 수 있습니다.

클래스 메서드 · 정적 메서드

객체를 만들지 않고 클래스 이름만으로 부를 수 있는 메서드도 있어요.

종류데코레이터특징
인스턴스 메서드없음첫 인자 self, 객체로 호출
클래스 메서드@classmethod첫 인자 cls, 클래스 변수 다룸
정적 메서드@staticmethodself·cls 없음, 독립 기능

4상속과 재정의

👨‍👦 상속(Inheritance)
부모 클래스의 속성과 메서드를 자식이 물려받는 것. 코드를 다시 안 쓰고 재사용할 수 있어요.
🧬 비유 — 유전이랑 같아요. "학생은 사람이다(Student is a Person)"처럼, 학생은 사람의 특징(이름·나이)을 물려받고 거기에 전공을 추가합니다.
python
class Person:
    def __init__(self, name, gender):
        self.name = name
        self.gender = gender
    def __str__(self):
        return f"{self.name}({self.gender})"

class Student(Person):            # Person을 상속
    def __init__(self, name, gender, major):
        super().__init__(name, gender)   # 부모 생성자 호출
        self.major = major               # 자식만의 속성 추가
    def __str__(self):                    # 재정의(오버라이딩)
        return super().__str__() + f", 전공:{self.major}"

s = Student("홍길동", "남자", "컴공")
print(s)
홍길동(남자), 전공:컴공
🔗 super()
부모 클래스를 가리켜요. super().__init__()으로 부모의 생성자를, super().메서드()로 부모의 메서드를 불러올 수 있어요.
✏️ 재정의(Overriding)
부모의 메서드를 자식에서 같은 이름으로 다시 정의하는 것. 함수 이름과 매개변수가 부모와 같아야 해요.
🎨 비유 — 물려받은 옷을 내 스타일로 수선하는 것과 같아요. 기본은 부모 것이지만, 자식이 자기 방식대로 바꿔 씁니다.
💡 __str__은 뭐예요?
객체를 print()할 때 어떻게 보여줄지 정하는 특별한 메서드예요. 재정의해두면 print(객체)가 사람이 읽기 좋은 문자열을 출력합니다.

정적 변수 (이름 맹글링)

변수 이름 앞에 밑줄 두 개 __를 붙이면, 파이썬이 내부적으로 이름을 바꿔(_클래스명__변수) 외부에서 함부로 못 건드리게 보호해요.

🎬 다음 화 예고 — 메뉴가 객체가 되어 가게가 제법 똑똑해졌어요. 마지막 관문이 남았습니다. 8장에서는 이상한 손님이 와도 가게가 멈추지 않도록 예외처리로 마무리합니다.
📝 연습문제
실습형
1도형(Shape)과 삼각형(Triangle) 클래스를 작성하세요.
· Shape: 생성자로 x,y 좌표(기본 0,0) 초기화, move(), __str__(), 정적 메서드 하나 이상
· Triangle은 Shape를 상속: width, height, x, y 초기화 / calc_area()(면적) / __str__() / 만들어진 객체 개수를 세는 클래스 변수(생성 시 +1, 소멸 시 -1)
정답
python
class Shape:
    def __init__(self, x=0, y=0):
        self.x = x
        self.y = y
    def move(self, dx, dy):
        self.x += dx
        self.y += dy
    def __str__(self):
        return f"좌표=({self.x},{self.y})"
    @staticmethod
    def info():
        return "도형 클래스입니다"

class Triangle(Shape):
    count = 0                       # 클래스 변수(모든 객체 공유)
    def __init__(self, width, height, x=0, y=0):
        super().__init__(x, y)      # 부모 생성자로 x,y 초기화
        self.width = width
        self.height = height
        Triangle.count += 1         # 객체 생길 때마다 +1
    def calc_area(self):
        return self.width * self.height / 2
    def __del__(self):
        Triangle.count -= 1         # 소멸될 때마다 -1
    def __str__(self):
        return f"삼각형 넓이={self.calc_area()}, {super().__str__()}"
    @classmethod
    def get_count(cls):
        return cls.count

t1 = Triangle(4, 3, 0, 10)
t2 = Triangle(3, 4, 5, 5)
print(t1); print(t2)
print('Triangle 객체 갯수 :', Triangle.get_count())
t1.move(50, 50)
print(t1)
삼각형 넓이=6.0, 좌표=(0,10) 삼각형 넓이=6.0, 좌표=(5,5) Triangle 객체 갯수 : 2 삼각형 넓이=6.0, 좌표=(50,60)
해설

핵심 포인트 3가지예요. ① 상속: Triangle(Shape)로 좌표 기능을 물려받고, super().__init__()으로 부모 생성자를 재활용. ② 클래스 변수 count: 모든 객체가 공유하므로 생성자에서 +1, 소멸자에서 -1 하면 현재 객체 수를 셀 수 있어요. ③ move(50,50)은 물려받은 메서드라 (0,10)→(50,60)으로 이동합니다.

서술형
2클래스와 객체(인스턴스)의 차이를 비유로 설명하세요.
예시 답안

클래스는 붕어빵 틀(설계도), 객체는 그 틀로 찍어낸 붕어빵이에요. 클래스는 "어떤 데이터와 기능을 가질지"를 정의한 틀일 뿐이고, 실제로 메모리에 만들어져 값을 담는 것이 객체(인스턴스)입니다. 하나의 클래스로 여러 객체를 만들 수 있어요.

3인스턴스 메서드에서 self는 무엇을 의미하나요?
정답·해설

self는 메서드를 호출한 객체 자기 자신을 가리켜요. p1.print_info()를 호출하면 self에 자동으로 p1이 들어가서, self.name은 곧 p1.name이 됩니다. 그래서 객체마다 다른 자기 데이터에 접근할 수 있어요.

4자식 클래스에서 부모의 do_(a) 메서드를 올바르게 호출하는 것은?
① Super.do_(self, a) ② super().do_(a) ③ super.do_(a)
정답

② super().do_(a)

해설

super()는 괄호를 붙여 호출해야 부모 클래스를 가리키는 객체가 돼요. ③ super(괄호 없음)는 클래스 자체라 틀리고, ① Super는 대문자 오타 + self를 직접 넘기는 옛날 방식이라 권장하지 않습니다.

5파이썬에서 생성자를 두 번 정의하면 어떻게 될까요?
정답·해설

마지막에 정의한 생성자만 유효해요. 파이썬은 생성자(함수) 중복 정의(오버로딩)를 지원하지 않아서, 같은 __init__을 두 번 쓰면 나중 것이 앞의 것을 덮어씁니다. 여러 방식으로 초기화하고 싶으면 기본값 매개변수나 클래스 메서드를 활용하세요.
