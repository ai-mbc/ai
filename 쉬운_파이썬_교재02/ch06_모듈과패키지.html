CHAPTER 06

모듈과 패키지

이제 '파이 분식' 코드가 길어졌어요. 계산 매뉴얼, 재고 매뉴얼, 손님 응대 매뉴얼… 한 파일에 다 넣으니 지저분합니다. 매뉴얼을 주제별 파일로 나누고(모듈), 남이 잘 만들어 둔 매뉴얼도 가져다 쓰는(import·pip) 법을 배워요.

🍜 파이 분식 이야기 — 6화. 매뉴얼을 서랍장에 정리하다

매뉴얼이 수십 장 쌓이자 뭐가 어디 있는지 모르겠어요. 사장님은 '계산 서랍', '재고 서랍', '응대 서랍'으로 나눠 정리합니다. 이렇게 나눈 파일 하나하나가 모듈, 서랍장 전체가 패키지예요. 게다가 세상엔 이미 전문가가 만들어 둔 매뉴얼(엑셀 계산, 그래프 그리기 등)이 무료로 널려 있어요. 그걸 pip로 받아 import로 꺼내 쓰면, 바퀴를 다시 발명할 필요가 없습니다.

🎯 이 장을 끝내면 이런 걸 할 수 있어요

  • 모듈을 import해서 남의 코드를 가져다 쓸 수 있다
  • import의 여러 방식을 구분해 쓸 수 있다
  • 내 코드를 모듈(.py 파일)로 만들 수 있다
  • 패키지가 무엇이고 어떻게 구성되는지 안다
  • pip으로 외부 패키지를 설치·삭제할 수 있다

1모듈이란?

📄 모듈(Module)
함수나 변수를 담아 놓은 파이썬 파일(.py) 하나예요. 자주 쓰는 코드를 파일로 만들어 두면 여러 곳에서 불러 쓸 수 있어요.
🗄️ 비유 — '파이 분식'의 매뉴얼 서랍 하나예요. 5장에서 만든 주문계산 함수를 calc.py라는 '계산 서랍'에 넣어 두면, 다른 파일에서 import calc 한 줄로 꺼내 씁니다. 매번 다시 만들 필요가 없죠.

파이썬에는 이미 만들어진 표준 모듈이 200개 넘게 있어요. 설치 없이 바로 import해서 씁니다.

python
import math          # 수학 모듈 가져오기
print(math.sqrt(16))  # 제곱근
print(math.pi)        # 원주율

import random
print(random.randint(1, 6))  # 주사위 굴리기
4.0 3.141592653589793 (1~6 중 랜덤한 값)
💡 대표적인 표준 모듈
math(수학) · random(난수) · datetime(날짜/시간) · os(운영체제) · sys(시스템) · json · sqlite3(DB) 등. 공식 문서: docs.python.org/3/library

2import 하는 여러 방법

모듈을 가져오는 방식이 몇 가지 있어요. 상황에 맞게 골라 씁니다.

방법뜻사용 예
import 모듈모듈 전체 가져오기math.sqrt(16)
from 모듈 import 함수특정 함수만 콕 집어sqrt(16) (모듈명 생략)
import 모듈 as 별칭짧은 별명 붙이기import numpy as np
from 모듈 import *모든 것 가져오기(비권장)—
python
from math import sqrt, pi   # 필요한 것만
print(sqrt(25), pi)

import random as rd          # 별칭
print(rd.randint(1, 10))
5.0 3.141592653589793 (1~10 중 랜덤)
⚠️ from 모듈 import *는 조심
모든 이름을 한꺼번에 가져오면 내 변수와 이름이 겹쳐 예상치 못한 문제가 생길 수 있어요. 되도록 필요한 것만 콕 집어 가져오는 게 좋습니다.
💡 dir() — 뭐가 들어있나 보기
dir(math)를 실행하면 그 모듈이 가진 함수·변수 목록을 볼 수 있어요.

3내 모듈 만들기

내 함수를 .py 파일로 저장하면, 그게 바로 나만의 모듈이에요.

📄 mymath.py
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

같은 폴더의 다른 파일에서 이렇게 불러 써요.

📄 main.py
import mymath

print(mymath.add(3, 5))
print(mymath.multiply(4, 2))
8 8
🚪 if __name__ == "__main__":
이 파일이 직접 실행될 때만 아래 코드를 돌려요. 다른 파일에서 import될 때는 실행되지 않습니다.
🎭 비유 — "내가 주인공으로 무대에 설 때만 이 대사를 하고, 조연(import)으로 불려 갔을 땐 조용히 있는다"는 스위치예요.
📄 mymath.py
def add(a, b):
    return a + b

if __name__ == "__main__":
    # 이 파일을 직접 실행할 때만 (테스트용)
    print("테스트:", add(1, 2))
💡 왜 쓸까?
모듈에는 함수만 담고 싶은데, 직접 실행해 테스트도 하고 싶을 때 유용해요. import될 땐 테스트 코드가 안 돌아가니 방해되지 않습니다.
  • sys.path — 파이썬이 모듈을 찾는 폴더 목록이에요. sys.path.append(경로)로 추가할 수 있어요.
  • __pycache__ — 모듈을 빠르게 불러오려고 만든 컴파일 캐시 폴더(.pyc 파일)예요. 자동 생성되니 신경 쓰지 않아도 됩니다.

4패키지(Package)

📦 패키지(Package)
여러 모듈을 폴더로 묶은 것이에요. 모듈이 파일이라면, 패키지는 그 파일들을 담은 폴더입니다.
🗄️ 비유 — 모듈이 '서류 한 장'이라면, 패키지는 서류들을 정리한 '폴더(서랍)'예요. 점(.)으로 폴더 안을 파고듭니다: 패키지.모듈

패키지는 이런 폴더 구조를 가져요.

sample_pac/ ├─ __init__.py └─ ab/ ├─ __init__.py ├─ a.py └─ b.py
💡 __init__.py의 역할
이 파일이 있으면 파이썬이 그 폴더를 "패키지"로 인식해요. 비어 있어도 괜찮아요. __all__ = [...]을 적으면 from 패키지 import * 할 때 어떤 모듈을 가져올지 지정할 수 있습니다.
python
from sample_pac.ab import a   # 패키지 안 모듈 가져오기
from sample_pac.ab.a import func  # 함수까지 직접
⚠️ 한 번 로드된 패키지는 다시 안 불러와요
수정한 모듈을 다시 반영하려면 커널을 재시작하거나 importlib.reload(모듈)을 써야 해요.

5외부 패키지 설치 (pip)

📥 pip
전 세계 개발자들이 만든 외부 패키지를 설치하는 도구예요. 앱스토어에서 앱 깔듯, 필요한 기능을 내려받습니다.
🛒 비유 — 온라인 쇼핑몰 같아요. pip install로 원하는 도구(numpy, pandas 등)를 '주문'하면 자동으로 설치됩니다.

명령창(터미널·Anaconda Prompt)에서 이렇게 써요.

bash (명령창)
pip install numpy          # 설치
pip install numpy==1.24.0   # 특정 버전 설치
pip show numpy              # 정보 보기
pip uninstall numpy         # 삭제

# 아나콘다에서는 conda도 사용
conda install pandas
conda list                  # 설치 목록
💡 표준 라이브러리 vs 외부 패키지
표준 라이브러리(math, random 등)는 파이썬에 기본 포함돼 설치가 필요 없어요. 반면 numpy, pandas 같은 외부 패키지는 pip으로 따로 설치해야 합니다. 외부 패키지 저장소는 pypi.org예요.
🎬 다음 화 예고 — 매뉴얼을 서랍장(모듈·패키지)에 정리했어요. 이제 메뉴 하나에 딸린 정보들을 한 덩어리로 묶고 싶어요. 7장에서는 메뉴를 객체(클래스)로 만들어 '살아 있는 물건'처럼 다룹니다.
📝 연습문제

모듈·패키지의 핵심을 확인하는 문제예요. (원본 교재의 연습문제 페이지를 대신해 구성했습니다)

1math 모듈을 가져와 100의 제곱근을 출력하세요.
정답
python
import math
print(math.sqrt(100))
10.0
해설

import math 후 math.sqrt()로 제곱근을 구해요. import 모듈 방식은 함수 앞에 모듈명을 붙여 씁니다.

2다음 두 코드의 차이를 설명하세요.
① import random   ② from random import randint
정답·해설

① 모듈 전체를 가져와서 random.randint(1,6)처럼 모듈명을 붙여 써요.
② randint 함수만 콕 집어 가져와서 randint(1,6)처럼 모듈명 없이 바로 씁니다. 필요한 것만 가져오니 간결하지만, 어느 모듈 것인지 드러나지 않는 단점도 있어요.

3numpy를 np라는 별칭으로 가져오는 코드를 작성하세요.
정답
python
import numpy as np
해설

as로 짧은 별명을 붙여요. 이후 np.array(...)처럼 씁니다. numpy→np, pandas→pd는 관례처럼 굳어진 별칭이에요.

4모듈(module)과 패키지(package)의 차이는 무엇일까요?
정답·해설

모듈은 함수·변수를 담은 파이썬 파일 하나(.py)이고, 패키지는 여러 모듈을 묶은 폴더예요. 비유하면 모듈은 '서류 한 장', 패키지는 '서류철(폴더)'. 패키지 안에는 보통 __init__.py 파일이 들어 있습니다.

5폴더가 '패키지'로 인식되도록 하려면 그 안에 어떤 파일이 있어야 할까요?
정답

__init__.py

해설

이 파일이 있으면 파이썬이 그 폴더를 패키지로 인식해요. 내용이 비어 있어도 됩니다. (최신 파이썬에선 없어도 인식되는 경우가 있지만, 명시적으로 두는 것이 안전해요.)

6if __name__ == "__main__": 구문은 언제 실행될까요?
정답·해설

그 파일을 직접 실행할 때만 실행돼요. 다른 파일에서 import로 불러올 때는 실행되지 않습니다. 그래서 모듈 안에 테스트 코드를 넣어 두는 용도로 많이 씁니다.

7requests라는 외부 패키지를 설치하는 명령을 작성하세요.
정답
bash
pip install requests
해설

명령창(터미널·Anaconda Prompt)에서 pip install 패키지명으로 설치해요. 삭제는 pip uninstall requests, 정보 확인은 pip show requests입니다.

① math ② numpy ③ random ④ pandas ⑤ datetime
정답

①, ③, ⑤ (math, random, datetime)

해설

①③⑤는 파이썬에 기본 포함된 표준 라이브러리라 바로 import돼요. ② numpy와 ④ pandas는 외부 패키지라서 pip install로 먼저 설치해야 합니다.
