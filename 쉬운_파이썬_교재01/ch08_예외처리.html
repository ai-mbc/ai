CHAPTER 08

예외처리

프로그램이 오류로 갑자기 멈추지 않도록, 미리 대비하는 방법을 배웁니다. 안정적인 프로그램의 필수 기술이에요.

🎯 이 장을 끝내면 이런 걸 할 수 있어요

  • 예외(오류)가 무엇인지 이해한다
  • try~except로 오류를 처리할 수 있다
  • 여러 예외를 구분해서 처리할 수 있다
  • raise로 예외를 일부러 발생시킬 수 있다
  • finally·with로 뒷정리를 안전하게 할 수 있다
  • 나만의 예외(사용자 정의 예외)를 만들 수 있다

1예외란 무엇일까?

⚠️ 예외(Exception)
프로그램 실행 중에 발생하는 오류예요. 문법은 맞지만, 돌리는 도중 예상 못한 상황을 만나 멈추는 것입니다.
🚗 비유 — 운전 중 갑자기 나타난 웅덩이 같아요. 미리 대비(예외처리)를 안 해두면 차가 멈춰버려요. 대비해두면 살짝 피해 계속 달릴 수 있죠.

이런 상황에서 예외가 잘 생겨요.

  • 파일이 없거나 쓰기 금지된 파일을 열려 할 때
  • 0으로 나눌 때 (ZeroDivisionError)
  • 리스트·튜플의 없는 인덱스를 참조할 때 (IndexError)
  • 네트워크 연결 실패, 데이터베이스 오류 등
💡 예외처리는 '복구'가 아니라 '대비'
예외처리는 시스템이 스스로 오류를 고치는 게 아니에요. 오류가 날 만한 곳을 미리 감싸서, 문제가 생겨도 프로그램이 계속 돌아가도록 대처 코드를 준비해 두는 것입니다.

2try ~ except

🛡️ try ~ except
try 블록에 위험한 코드를 넣고, 오류가 나면 except 블록이 대신 실행돼요. 오류가 없으면 except는 그냥 건너뜁니다.
🪂 비유 — 낙하산 같아요. try는 뛰어내리는 것, except는 문제가 생겼을 때 펴지는 비상 낙하산. 무사하면 낙하산은 안 펴집니다.
python
try:
    num = int(input("정수를 입력: "))   # 위험 구간
    print(100 / num)
except ValueError:
    print("정수가 아니에요!")            # 변환 실패 시
except ZeroDivisionError:
    print("0으로 나눌 수 없어요!")        # 0 입력 시
입력 "abc" → 정수가 아니에요! 입력 "0" → 0으로 나눌 수 없어요!
⚠️ 예외 이름을 지정하는 게 좋아요
except:처럼 이름을 생략하면 모든 오류를 뭉뚱그려 잡아요. 어떤 오류인지 알 수 없어 디버깅이 어렵습니다. except ValueError:처럼 구체적으로 지정하세요.

여러 예외를 구분해서 처리

예외마다 다르게 대응하려면 except를 여러 개 쓰거나, 튜플로 묶어요.

python
try:
    L = [1, 2, 3]
    print(L[10])
except (IndexError, KeyError) as e:   # 여러 예외를 한 번에
    print("인덱스/키 오류:", e)         # as e 로 오류 정보 받기
인덱스/키 오류: list index out of range
⚠️ 순서 규칙 — 자식 예외를 먼저!
여러 except를 쓸 때 부모(상위) 예외를 먼저 쓰면 그 아래 except는 영영 실행되지 않아요. Exception(모든 예외의 부모)처럼 넓은 것은 맨 마지막에 둬야 합니다.

else 절

else는 오류가 안 났을 때만 실행돼요. (except 다음에 위치)

python
try:
    result = 10 / 2
except ZeroDivisionError:
    print("오류!")
else:
    print("성공:", result)   # 오류 없을 때만
성공: 5.0

3raise — 예외를 일부러 발생시키기

🚨 raise
프로그래머가 직접 예외를 강제로 일으키는 명령이에요. "이건 잘못된 값이야!"라고 알릴 때 씁니다.
🙋 비유 — 심판이 반칙에 휘슬을 부는 것과 같아요. 규칙에 어긋나면 스스로 "여기 문제 있어요!"라고 신호를 보냅니다.
python
def divide(a, b):
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다")   # 강제 발생
    return a / b

try:
    divide(10, 0)
except ValueError as e:
    print("오류 잡음:", e)
오류 잡음: 0으로 나눌 수 없습니다
💡 추상 클래스에도 쓰여요
파이썬엔 추상 클래스 전용 키워드가 없어서, 자식이 반드시 재정의해야 하는 메서드에 raise NotImplementedError를 넣어 둬요. 자식이 구현 안 하고 부르면 오류로 알려주는 방식입니다.

4사용자 정의 예외

🏷️ 사용자 정의 예외
파이썬 기본 예외로 부족할 때, Exception을 상속받아 나만의 예외를 만들 수 있어요.
🎫 비유 — 우리 회사만의 '전용 경고 스티커'를 만드는 것과 같아요. 기성품(기본 예외)으로 표현 안 되는 상황을 내 이름으로 표시합니다.
python
class EmptyDataError(Exception):     # Exception 상속
    pass

def process(data):
    if len(data) == 0:
        raise EmptyDataError("데이터가 비었습니다")
    return sum(data)

try:
    process([])
except EmptyDataError as e:
    print("내 예외 발생:", e)
내 예외 발생: 데이터가 비었습니다

5정리 작업 (finally / with)

🧹 finally
오류가 나든 안 나든 항상 실행되는 블록이에요. 파일 닫기, 연결 해제 같은 뒷정리에 씁니다.
🔚 비유 — 영화가 어떻게 끝나든 반드시 나오는 엔딩 크레딧 같아요. 성공하든 실패하든 마지막 정리는 꼭 합니다.
python
try:
    f = open("data.txt")
    print(f.read())
except FileNotFoundError:
    print("파일이 없어요")
finally:
    print("정리 작업 (항상 실행)")
파일이 없어요 정리 작업 (항상 실행)
📎 with — 자동 정리
파일 같은 자원을 다 쓰면 자동으로 닫아주는 편리한 구문이에요. finally로 일일이 닫을 필요가 없어요.
python
# with 블록이 끝나면 파일 f가 자동으로 닫힘
with open("data.txt", "w") as f:
    f.write("안녕")
# 여기서 파일은 이미 안전하게 닫혀 있음
💡 with를 추천하는 이유
파일을 열고 close()를 깜빡하면 자원이 낭비돼요. with는 블록을 벗어나는 순간 알아서 닫아줘서 실수를 막아줍니다. 파일 다룰 땐 with가 정석이에요.
📝 연습문제
실습형
1사용자에게 숫자 두 개를 입력받아 나눗셈 결과를 출력하되, 0으로 나누거나 숫자가 아닌 값을 넣으면 각각 알맞은 메시지를 출력하세요.
정답
python
try:
    a = int(input("첫 번째 수: "))
    b = int(input("두 번째 수: "))
    print("결과:", a / b)
except ValueError:
    print("숫자를 입력하세요!")
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다!")
해설

두 종류의 오류를 따로 잡아요. 글자를 넣으면 int()에서 ValueError, 0을 넣으면 나눗셈에서 ZeroDivisionError가 발생합니다. 예외를 구체적으로 지정하면 상황별로 친절한 안내가 가능해요.

2나이가 음수이면 ValueError를 발생시키는 함수 set_age(age)를 작성하고, 호출부에서 예외를 처리하세요.
정답
python
def set_age(age):
    if age < 0:
        raise ValueError("나이는 음수일 수 없습니다")
    return age

try:
    set_age(-5)
except ValueError as e:
    print("오류:", e)
오류: 나이는 음수일 수 없습니다
해설

잘못된 값을 스스로 걸러내 raise로 알리는 패턴이에요. 값을 검증(validation)할 때 아주 흔하게 씁니다.

문제풀이형
3예외 처리에 대해 잘못 설명한 것은?
① try 블록은 예외 발생 가능성이 있는 문장을 작성한다 ② 예외가 발생하면 except 블록이 실행된다 ③ 상위 예외 except 블록이 하위 예외보다 먼저 선언되어야 한다 ④ finally 블록은 예외 발생 유무와 상관없이 실행된다
정답

③

해설

반대예요! 하위(구체적) 예외를 먼저, 상위(넓은) 예외를 나중에 써야 해요. Exception 같은 상위 예외를 먼저 두면 모든 오류를 거기서 잡아버려, 아래 except는 실행될 기회가 없어집니다.

4예외 처리에 대한 설명 중 잘못된 것은?
① raise는 강제로 예외를 발생시킨다 ② catch 절은 예외를 처리하는 구문이다 ③ 여러 예외를 하나의 구문으로 처리할 수 있다 ④ else 절은 예외가 없을 때 실행된다
정답

②

해설

파이썬은 catch가 아니라 except를 써요! catch는 자바·C++에서 쓰는 키워드입니다. 파이썬 예외처리 키워드는 try / except / else / finally예요.

5다음 중 except 절을 잘못 사용한 것은?
① except : ② except Exception: ③ except Exception as e: ④ except Exception e:
정답

④

해설

예외 객체를 변수로 받으려면 반드시 as를 써야 해요: except Exception as e:. except Exception e:처럼 as 없이 쓰면 문법 오류입니다. ①②③은 모두 올바른 형태예요.

6다음 중 예외 처리에 사용하지 않는 구문은?
① try ② except ③ with ④ finally
정답

③ with

해설

예외 처리의 핵심 구문은 try / except / else / finally예요. with는 자원 정리(파일 자동 닫기)에 쓰는 별개의 구문입니다. (정리 작업이라는 점에서 finally와 목적이 비슷하지만, try-except 구조의 일부는 아니에요.)

7finally 블록과 else 블록의 실행 조건 차이는 무엇일까요?
정답·해설

else는 예외가 발생하지 않았을 때만 실행돼요. 반면 finally는 예외 발생 여부와 상관없이 항상 실행됩니다. 그래서 else는 '성공 시 추가 작업', finally는 '무조건 해야 할 뒷정리'에 씁니다.
