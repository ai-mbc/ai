CHAPTER 05

함수

반복되는 코드를 하나의 이름으로 묶어 재사용하는 방법을 배웁니다. 함수를 쓰면 코드가 짧고 깔끔해져요.

🎯 이 장을 끝내면 이런 걸 할 수 있어요

  • 함수를 정의(def)하고 호출할 수 있다
  • 매개변수와 인수의 차이를 안다
  • return으로 결과를 돌려받을 수 있다
  • 지역변수와 전역변수를 구분할 수 있다
  • 기본값·키워드 인수·*args·**kwargs를 활용할 수 있다
  • 람다식으로 짧은 함수를 만들 수 있다

1함수란 무엇일까?

🛠️ 함수(Function)
자주 쓰는 코드를 하나로 묶어 이름을 붙인 것이에요. 값을 넣으면(입력) 처리해서 결과를 내놓아요(출력).
🥤 비유 — 자판기 같아요. 동전(입력)을 넣고 버튼을 누르면 음료(출력)가 나와요. 안에서 어떻게 만드는지 몰라도 쓸 수 있죠. 게다가 몇 번이든 다시 쓸 수 있어요.

함수는 두 단계로 이뤄져요.

  • 정의(define) — 함수를 def로 만드는 것 (자판기 설치)
  • 호출(call) — 만든 함수를 이름()으로 실행하는 것 (버튼 누르기)
python
def greet():              # 정의
    print("안녕하세요!")

greet()   # 호출
greet()   # 몇 번이든 재사용
안녕하세요! 안녕하세요!

매개변수와 인수

📥 매개변수(Parameter) vs 인수(Argument)
매개변수는 함수를 정의할 때 값을 받으려고 만든 빈 상자, 인수는 호출할 때 실제로 넣는 값이에요.
📮 비유 — 매개변수는 우체통(받을 곳), 인수는 그 안에 넣는 편지(실제 값)라고 생각하세요.
python
def add(a, b):        # a, b는 매개변수
    print(a + b)

add(3, 5)             # 3, 5는 인수
8
⚠️ 개수를 맞춰야 해요
매개변수가 2개면 인수도 2개를 넘겨야 해요(기본값이 없는 한). 개수가 안 맞으면 에러가 납니다.
💡 docstring — 함수 설명서
함수 첫 줄에 """설명"""을 적으면 함수 사용법 메모가 돼요. help(함수명)으로 볼 수 있습니다.

2return — 결과 돌려주기

📤 return
함수가 계산한 결과를 밖으로 돌려줘요. print는 화면에 보여줄 뿐이지만, return은 값을 가져와 다시 쓸 수 있게 합니다.
python
def add(a, b):
    return a + b        # 결과를 돌려줌

result = add(3, 5)      # 돌려받은 값을 저장
print(result * 2)       # 다시 활용 가능
16
⚠️ print와 return은 달라요
print는 화면에 보여만 주고, return은 값을 돌려줘서 재사용하게 해요. return이 없는 함수를 변수에 담으면 None(아무것도 없음)이 저장됩니다.

여러 개 값 반환하기

콤마로 나열하면 여러 값을 한 번에 돌려줘요. 이때 값들은 튜플로 묶입니다.

python
def min_max(data):
    return min(data), max(data)

low, high = min_max([3, 7, 1, 9])   # 각각 받기
print(low, high)
1 9

3지역변수와 전역변수

🏠 지역변수 (Local) vs 🌍 전역변수 (Global)
지역변수는 함수 안에서만 사는 변수, 전역변수는 함수 밖에서 만들어 어디서나 볼 수 있는 변수예요.
🏘️ 비유 — 지역변수는 '집 안에서만 쓰는 물건', 전역변수는 '동네 공용 물건' 같아요. 집 안 물건은 밖에서 못 꺼내 씁니다.
python
num = 20              # 전역변수

def show():
    print(num)        # 전역변수 읽기는 OK → 20

def local_test():
    x = 10            # 지역변수 (이 함수 안에서만)
    print(x)

show()
local_test()
# print(x)  → 에러! x는 함수 밖에서 안 보임
20 10
⚠️ 함수 안에서 전역변수를 바꾸려면?
함수 안에서 전역변수에 새 값을 대입하려면 global 키워드가 필요해요. 안 그러면 같은 이름의 지역변수가 새로 생겨버립니다.
python
count = 0
def increase():
    global count      # "전역 count를 쓸게" 선언
    count += 1

increase()
print(count)          # 1
1
💡 변수를 찾는 순서
파이썬은 변수를 지역 → 전역 → 내장 순서로 찾아요. 함수 안에서 이름을 부르면 먼저 자기 집(지역)을 뒤지고, 없으면 동네(전역)를 봅니다.

4값에 의한 호출 vs 참조에 의한 호출

함수에 값을 넘길 때, 자료형에 따라 동작이 달라요. 이게 초보자에게 헷갈리는 부분이에요.

📄 값에 의한 호출 (숫자·문자열·튜플)
함수 안에서 값을 바꿔도 원본은 그대로예요. 복사본을 넘긴 것과 같아요.
python
var = 100
def change(var):
    var = 200          # 함수 안의 var만 바뀜
change(var)
print(var)             # 여전히 100!
100
📋 참조에 의한 호출 (리스트·딕셔너리)
함수 안에서 내용을 바꾸면 원본도 바뀌어요. 같은 그릇을 함께 보고 있기 때문이에요.
python
L = [1, 2, 3]
def add_item(lst):
    lst.append(4)      # 원본 리스트를 직접 수정
add_item(L)
print(L)               # [1, 2, 3, 4] 로 바뀜!
[1, 2, 3, 4]
💡 한 줄 정리
바꿀 수 없는 값(숫자·문자열·튜플)은 원본 안전, 바꿀 수 있는 값(리스트·딕셔너리)은 원본까지 영향. 이 차이를 알면 예상치 못한 버그를 피할 수 있어요.

5다양한 매개변수

기본값을 갖는 매개변수

매개변수에 기본값을 정해두면, 인수를 생략했을 때 그 값이 쓰여요.

python
def greet(name, greeting="안녕"):
    print(f"{greeting}, {name}!")

greet("철수")               # 기본값 사용
greet("영희", "반가워")     # 직접 지정
안녕, 철수! 반가워, 영희!

키워드 인수

매개변수 이름을 직접 지정해서 넘기면 순서를 바꿔도 돼요.

python
def profile(name, age):
    print(name, age)

profile(age=20, name="홍길동")   # 순서 바꿔도 OK
홍길동 20
⚠️ 순서 규칙
키워드 인수(name=값)는 순서 인수 뒤에 와야 해요. profile("홍길동", age=20)은 되지만 profile(name="홍길동", 20)은 에러입니다.

*args 와 **kwargs — 개수가 정해지지 않은 인수

📦 *args (가변 위치 인수)
인수를 몇 개든 받아 튜플로 묶어줘요. 매개변수 앞에 *를 붙입니다.
🏷️ **kwargs (가변 키워드 인수)
이름=값 형태를 몇 개든 받아 딕셔너리로 묶어줘요. **를 붙입니다.
python
def total(*args):          # 여러 숫자를 튜플로
    return sum(args)

def info(**kwargs):        # 여러 쌍을 딕셔너리로
    for key, value in kwargs.items():
        print(key, ":", value)

print(total(1, 2, 3, 4))
info(name="홍길동", age=20)
10 name : 홍길동 age : 20
💡 언패킹(Unpacking) — 반대로 풀기
이미 리스트/딕셔너리에 담긴 값을 함수에 넘길 땐 *, **로 풀어줘요. add(*[3, 5])는 add(3, 5)와 같아요.

6람다식 — 이름 없는 짧은 함수

⚡ 람다(lambda)
이름 없이 한 줄로 만드는 작은 함수예요. return 없이도 결과를 돌려줍니다. 문장 하나만 담을 수 있어요.
📝 비유 — 정식 함수가 '이름 붙인 레시피 책'이라면, 람다는 '포스트잇에 급히 적은 간단 메모' 같아요.
python
# 일반 함수
def square(x):
    return x ** 2

# 같은 걸 람다로
square2 = lambda x: x ** 2

print(square(4), square2(4))
16 16

람다는 map(), filter(), sorted()처럼 함수를 인수로 받는 곳에서 특히 빛나요.

python
nums = [1, 2, 3, 4, 5]
# 각 값을 제곱 (map)
print(list(map(lambda x: x**2, nums)))
# 짝수만 걸러내기 (filter)
print(list(filter(lambda x: x%2==0, nums)))
[1, 4, 9, 16, 25] [2, 4]

def get_max_min(data_list):

2체중(kg)과 신장(m)을 받아 BMI 상태를 출력하는 함수를 작성하세요. (BMI = 체중 ÷ 신장², 18.5 미만 마른체형 / 25 미만 표준 / 30 미만 비만 / 이상 고도비만)
정답
python
def getBMI(kg, m):
    bmi = kg / (m ** 2)
    if bmi < 18.5:
        return "마른체형"
    elif bmi < 25:
        return "표준"
    elif bmi < 30:
        return "비만"
    else:
        return "고도비만"

print(getBMI(70, 1.75))
표준
해설

제곱은 m ** 2. 조건은 낮은 값부터 elif로 이어가면 범위가 자동으로 나뉘어요. 70÷(1.75²)≈22.9라서 '표준'입니다.

4시작값과 끝값을 받아 그 사이 모든 정수의 합을 리턴하는 함수를 작성하세요. (양끝 포함)
정답
python
def mysum(start, end):

print(mysum(1, 10))
55
해설

range는 끝값을 포함하지 않으므로 +1을 해줘야 end까지 더해져요. 1~10 합은 55입니다.

5문자열 리스트를 받아 각 문자열의 첫 세 글자로 이뤄진 리스트를 리턴하세요. 예: ['Seoul','Daegu','Kwangju','Jeju'] → ['Seo','Dae','Kwa','Jej']
정답 (함수 & 람다 두 방식)
python
# 함수 + 리스트 컴프리헨션
def get_abbrs(lst):
    return [s[:3] for s in lst]

# 람다 + map
get_abbrs2 = lambda lst: list(map(lambda s: s[:3], lst))

print(get_abbrs(['Seoul','Daegu','Kwangju','Jeju']))
['Seo', 'Dae', 'Kwa', 'Jej']
해설

s[:3]은 앞 세 글자를 잘라내는 슬라이싱이에요. 각 문자열에 이걸 적용하면 됩니다.

6ex = [1,2,3,4,5]를 [1,4,9,16,25]로 바꾸세요. (① 람다+map ② 리스트 컴프리헨션)
정답
python
ex = [1, 2, 3, 4, 5]
print(list(map(lambda x: x**2, ex)))   # ① map
print([x**2 for x in ex])              # ② 컴프리헨션
[1, 4, 9, 16, 25] [1, 4, 9, 16, 25]
해설

각 값을 제곱하는 작업이에요. map은 "각 항목에 함수 적용", 리스트 컴프리헨션은 "한 줄로 리스트 생성" — 결과는 같습니다.

7여러 벡터(리스트)들의 크기가 모두 같은지 True/False로 리턴하는 vector_size_check(*vec)를 한 줄로 작성하세요.
정답
python
def vector_size_check(*vec):
    return len(set(len(v) for v in vec)) == 1

print(vector_size_check([1,2,3], [4,5,6], [7,8,9]))
print(vector_size_check([1,2], [1,2,3]))
True False
해설

각 벡터의 길이를 모아 set으로 만들면 중복이 제거돼요. 길이가 다 같으면 set에 원소가 1개만 남으니, 개수가 1인지 확인하면 됩니다. *vec로 벡터를 몇 개든 받는 게 포인트!
